Section 06

Exploring a degree, filling out a form, making a gift, looking for help. Each of these is someone's experience of UCF. The digital brand isn't styling on separate sites. It's whether the experience is clear, useful and trustworthy every time.

  • Clear. Structured, predictable, easy to scan.
  • Useful. Answer the question they arrived with, then surface the next step.
  • Accessible. WCAG 2.2 AA minimum, every device, every ability.
  • Consistent. Same identity, navigation and trust signals across UCF.
  • Trustworthy. Current information, clear ownership, defensible claims.
  • They didn't start here. Most people land deep in the site from search. Every page is a front door.
  • 58% of UCF web sessions start at Google — not the homepage.
  • 50%+ of UCF web visits happen on a phone.
  • A rising share of traffic comes from AI answer engines.
  • WCAG 2.2 AA accessibility is the floor for every UCF page.

Preview every page on your phone before your desktop. If the point isn't in the first screen on mobile, move it up — that's the screen most people actually get.

Matthew W. — Web Lead, Web Communications

Shared System

Design within the UCF system.

  • Use the shared UCF header. Required on every official page — no exceptions, no custom rebuilds. Instructions available at universityheader.ucf.edu.
  • Use approved templates and components. Reuse nav, buttons and page structures. Consistency and accessibility come baked in.
  • Follow the UCF design system. The component library is evolving. Check go.ucf.edu/web before building something new.

Mobile

Mobile is the default.

Above the fold — desktop and mobile

Nav / logo, one clear H1, meta description, a subhead that adds context, one primary CTA. If it doesn't fit the first screen, it isn't the priority. Write the first screen like it's the only one you get.

Web Writing

Writing for the web.

  • Design for every visitor. WCAG 2.2 AA is the floor, not the ceiling. Contrast, keyboard access, alt text, structure — from the first draft.
  • Speed is a first impression. Size and compress every image for the screen it loads on.
  • Headlines that stand alone. Name the program, the place, the result. It should make sense pulled into a search result.
  • Structure is what machines read. Real lists, real tables, real headings. Link text that describes where it goes — never "click here."
  • Write for AI citations & Google. Lead with the direct answer. State the fact plainly. Keep it current and sourced (E-E-A-T).
  • Know why they came — and measure it. Every UCF page needs Analytics and Search Console connected at launch. See Measurement.
